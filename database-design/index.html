项目概述

1.1 项目背景

物资管理是企业日常运营中的核心环节。传统的手工管理方式存在效率低、易出错、信息不透明等问题。本系统旨在通过信息化手段，实现对物资的入库、出库、库存、资金等全流程管理，提高管理效率和准确性。

1.2 系统目标

  • 实现物资信息的集中化管理
  • 实现入库、出库操作的自动化库存更新
  • 实现资金下拨与入账的闭环管理
  • 支持管理员与普通用户的分级权限管理
  • 提供多维度报表统计与分析功能

1.3 系统角色

角色 职责
管理员 物资管理、入库/出库管理、资金管理、计划管理、参数维护、用户管理、报表查看
普通用户 物资查询、资金查询、密码修改

5.2 数据库连接层设计

config/db_config.py 负责统一管理数据库连接，采用 pyodbc 驱动，支持 SQL Server 身份验证。

import pyodbc

class DBConfig:
    SERVER = '127.0.0.1'
    DATABASE = '物资管理系统'
    UID = 'sa'
    PWD = '123456'
    DRIVER = '{SQL Server Native Client 10.0}'

    @classmethod
    def get_connection(cls):
        conn_str = (
            f"DRIVER={cls.DRIVER};"
            f"SERVER={cls.SERVER};"
            f"DATABASE={cls.DATABASE};"
            f"UID={cls.UID};"
            f"PWD={cls.PWD};"
        )
        return pyodbc.connect(conn_str)

class LoginWindow(QWidget):
    def handle_login(self):
        username = self.username_input.text().strip()
        password = self.password_input.text().strip()

        user = UserModel.login(username, password)
        if user:
            # 根据角色跳转不同主窗口

                self.main_window = UserMainWindow(user)

用户模型 (models/user_model.py)

class UserModel(BaseModel):
    table_name = 'Users'
    primary_key = 'UserID'

    @classmethod
    def login(cls, username, password):
        sql = "SELECT * FROM Users WHERE Username = ? AND Password = ?"
        results = cls.execute_query(sql, (username, password))
        return results[0] if results else None

    @classmethod
    def register(cls, username, password, role='普通用户'):
        if cls.username_exists(username):
            return False
        sql = "INSERT INTO Users (Username, Password, UserRole) VALUES (?, ?, ?)"
        return cls.execute_non_query(sql, (username, password, role)) > 0

6.2 物资管理模块

物资模型 (models/material_model.py)

class MaterialModel(BaseModel):
    table_name = 'Materials'
    primary_key = 'MaterialID'

    @classmethod
    def search_by_name(cls, keyword):
        sql = "SELECT * FROM Materials WHERE MaterialName LIKE ?"
        return cls.execute_query(sql, (f'%{keyword}%',))

    @classmethod
    def update_stock(cls, material_id, quantity_change):
        material = cls.get_by_id(material_id)
        if not material:
            return False
        new_quantity = material['StockQuantity'] + quantity_change
        if new_quantity < 0:
            return False
        sql = "UPDATE Materials SET StockQuantity = ? WHERE MaterialID = ?"
        return cls.execute_non_query(sql, (new_quantity, material_id)) > 0

6.3 入库管理模块

入库模型 (models/stock_model.py - StockInModel)

class StockInModel(BaseModel):
    table_name = 'StockIn'
    primary_key = 'InID'

    @classmethod
    def add_stock_in(cls, material_id, quantity, operator, in_time=None):
        # 事务处理：插入入库记录 + 更新库存
        conn = BaseModel.get_connection()
        try:
            cursor = conn.cursor()
            sql_in = "INSERT INTO StockIn (MaterialID, InQuantity, Operator) VALUES (?, ?, ?)"
            cursor.execute(sql_in, (material_id, quantity, operator))
            sql_update = "UPDATE Materials SET StockQuantity = StockQuantity + ? WHERE MaterialID = ?"
            cursor.execute(sql_update, (quantity, material_id))
            conn.commit()
            return True
        except Exception:
            conn.rollback()
            return False
        finally:
            conn.close()

6.4 出库管理模块

出库模型 (models/stock_model.py - StockOutModel)

class StockOutModel(BaseModel):
    table_name = 'StockOut'
    primary_key = 'OutID'

    @classmethod
    def add_stock_out(cls, material_id, dept_id, quantity, operator):
        # 先检查库存是否充足
        material = MaterialModel.get_by_id(material_id)
        if not material or material['StockQuantity'] < quantity:
            return False

        # 事务处理：插入出库记录 + 扣减库存
        conn = BaseModel.get_connection()
        try:
            cursor = conn.cursor()
            sql_out = "INSERT INTO StockOut (MaterialID, DeptID, OutQuantity, Operator) VALUES (?, ?, ?, ?)"
            cursor.execute(sql_out, (material_id, dept_id, quantity, operator))
            sql_update = "UPDATE Materials SET StockQuantity = StockQuantity - ? WHERE MaterialID = ?"
            cursor.execute(sql_update, (quantity, material_id))
            conn.commit()
            return True
        except Exception:
            conn.rollback()
            return False
        finally:
            conn.close()

class ReportModel:
    @staticmethod
    def get_fund_summary():
        sql = """
            SELECT IncomeDate AS FlowDate, '收入' AS FlowDirection, 
                   Amount AS FlowAmount, Source AS SourcePurpose
            FROM FundIncomes
            UNION ALL
            SELECT AllocationDate AS FlowDate, '支出' AS FlowDirection,
                   -Amount AS FlowAmount, Purpose AS SourcePurpose
            FROM FundAllocations
            ORDER BY FlowDate DESC
        """
        return BaseModel.execute_query(sql)

开发过程中遇到的问题与解决方案

问题 1：PyQt5 无法启动 - "no Qt platform plugin"

现象：

This application failed to start because no Qt platform plugin could be initialized.

原因：PyQt5 找不到 Qt 插件路径，通常是因为插件路径未正确设置。

解决方案：在 main.py 中手动设置环境变量。

import os
# 自动定位 PyQt5 的 plugins 目录
pyqt5_path = os.path.dirname(PyQt5.__file__)
plugin_path = os.path.join(pyqt5_path, 'Qt5', 'plugins')
os.environ['QT_QPA_PLATFORM_PLUGIN_PATH'] = plugin_path

问题 2：SQL Server 连接失败 - "Adaptive Server connection failed"

现象：

DB-Lib error message 20009: Unable to connect: Adaptive Server is unavailable

原因： - SQL Server 未启用 TCP/IP 协议 - TCP/IP 端口未正确配置 - SQL Server Browser 服务未启动（非必需，但对于固定端口可绕过）

解决方案： 1. 在 SQL Server 配置管理器中启用 TCP/IP 协议 2. 启用 IP1（127.0.0.1），设置 TCP 端口 = 1433 3. 重启 SQL Server 服务 4. 使用 服务器名,端口号 格式连接

问题 3：Windows 身份验证失败 - "该登录名来自不受信任的域"

现象：

登录失败。该登录名来自不受信任的域，不能与 Windows 身份验证一起使用。

原因：SQL Server 配置为仅允许域账户，本地账户不被信任。

解决方案：启用 SQL Server 身份验证，使用 sa 账户登录。

问题 4：PyQt5 安装到用户目录导致插件路径不一致

现象：PyQt5 被安装到 C:\Users\用户名\AppData\Roaming\Python\，而非系统 Python 目录。

原因：系统 Python 目录无写入权限，pip 自动降级到用户目录。

解决方案：

import PyQt5
import os
pyqt5_path = os.path.dirname(PyQt5.__file__)
plugin_path = os.path.join(pyqt5_path, 'Qt5', 'plugins')
os.environ['QT_QPA_PLATFORM_PLUGIN_PATH'] = plugin_path

问题 5：报表模块因 None 值导致格式化错误

现象：

unsupported format string passed to NoneType.__format__

原因：数据库查询结果中存在 None 值，f"{value:.2f}" 格式化时出错。

解决方案：在显示前做空值判断。

amount = record.get('FlowAmount')
if amount is None:
    amount = 0
amount_str = f"{amount:.2f}"

本次课程设计完成了一个功能完整的物资管理信息系统，涵盖了从需求分析、数据库设计、系统开发到测试部署的全流程。系统实现了以下核心功能：

  • ✅ 用户登录/注册与权限管理
  • ✅ 物资信息的增删改查
  • ✅ 入库与出库的自动化库存更新
  • ✅ 资金下拨与入账管理
  • ✅ 需求计划、库存计划、采购计划管理
  • ✅ 多维度报表统计与分析
  • ✅ 管理员与普通用户的差异化界面

8.2 技术心得

  1. MVC 架构的优势：分层设计使代码结构清晰，便于维护和扩展。模型层负责数据操作，视图层负责界面展示，控制器层负责业务逻辑，三者各司其职。

  2. 事务处理的重要性：入库和出库操作涉及多条 SQL 语句（插入记录 + 更新库存），必须使用事务保证数据一致性。conn.commit() 和 conn.rollback() 的配合使用是关键。

  3. 参数化查询的必要性：使用 ? 占位符配合参数元组执行 SQL 查询，可以有效防止 SQL 注入攻击。

  4. 数据库连接管理的规范性：统一在 db_config.py 中管理连接配置，所有模型通过 BaseModel 获取连接，避免代码重复。

8.3 经验教训

  1. 环境配置要细致：PyQt5 的插件路径、SQL Server 的 TCP/IP 配置、sa 账户的启用等细节问题需要逐一排查。

  2. 异常处理要全面：数据库操作可能因各种原因失败（连接断开、数据异常、约束冲突等），需要在代码中做好异常捕获和友好提示。

  3. 界面与业务分离：一开始如果能把界面代码和业务逻辑分开，后期的功能添加和修改会轻松很多。

8.4 未来扩展方向

A. 建表 SQL

-- 创建数据库
CREATE DATABASE 物资管理系统;
GO

USE 物资管理系统;
GO

-- Users 用户表
CREATE TABLE Users (
    UserID INT IDENTITY(1,1) PRIMARY KEY,
    Username NVARCHAR(50) UNIQUE NOT NULL,
    Password NVARCHAR(50) NOT NULL,
    UserRole NVARCHAR(20) DEFAULT '普通用户',
    CreateDate DATETIME DEFAULT GETDATE()
);

-- Departments 部门表
CREATE TABLE Departments (
    DeptID INT IDENTITY(1,1) PRIMARY KEY,
    DeptName NVARCHAR(50) NOT NULL,
    FundAmount DECIMAL(18,2) DEFAULT 0,
    Remark NVARCHAR(200)
);

-- Materials 物资表
CREATE TABLE Materials (
    MaterialID INT IDENTITY(1,1) PRIMARY KEY,
    MaterialName NVARCHAR(100) NOT NULL,
    UnitPrice DECIMAL(18,2) DEFAULT 0,
    Unit NVARCHAR(20),
    Supplier NVARCHAR(100),
    Description NVARCHAR(500),
    StockQuantity INT DEFAULT 0
);
